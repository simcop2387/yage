class Image ;
A class for loading and manipulating images. Supports loading images from any format supported by SDL_Image. Currently supports grayscale, RGB, and RGBA image data.

BUGS:
An RGB image will often be returned when loading grayscale images. Use setFormat(IMAGE_FORMAT_GRAYSCALE) to correct this. The load and resize functions seem to have images with widths that aren't a multiple of 4.

this(char[] filename);
Construct and load image data from filename.

this(ubyte[] image, uint width, uint height, int format);
Construct from image data in memory

Params:
ubyte[] image a 1-dimensional array of raw image data
uint width height in pixels
uint height height in pixels
int format one of the IMAGE_FORMAT_* constants from yage.system.constant

ubyte[] get ();
Return the raw image data. Row-major order is used. This means that the array contains all of row 1's pixels, followed by row 2's pixels, etc.

int getFormat ();
Get the format of the image.

See Also:
The IMAGE_FORMAT_* constants in system.constant.

int getHeight ();
Get the height of the image in pixels.

char[] getSource ();
Get the name of the file the image was loaded from, or an empty string if loaded from memory.

int getWidth ();
Get the width of the image in pixels.

ubyte[] opIndex (uint i);
Get the ith pixel in the image.

ubyte[] opIndex (uint x, uint y);
Get the pixel at the given coordinates.

void* ptr ();


void resize (int new_width, int new_height);
Resize the image via glu.

BUGS:
This function often produces weird results if new_width is not a multiple of 4.

void setFormat (int format);
Set the format of the image data. This is an in-place operation.

Params:
int format one of the IMAGE_FORMAT_* constants from yage.system.constant

Image subImage (int top, int left, int bottom, int right);
Return a new image that is a sub-image of this image. The four parameters should be in pixels. The sub-image will start with the pixels including top and left and stop just before pixels specified by bottom and right.
